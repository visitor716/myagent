# Agent Memory

This directory is the file-backed long-term memory for local Codex work.

Codex should update these files when a stable preference, recurring workflow,
project decision, or open loop appears. Keep entries short, dated, and free of
secrets. Prefer links to concrete files or commands over vague summaries.

## Files

- `codex-operating-memory.md` - durable user preferences and runtime rules.
- `open-loops.md` - active work that should survive context compaction.
- `decisions.md` - decisions worth preserving so future agents do not
  rediscover them.
- `heartbeat.md` - latest automated local status snapshot, generated by
  `scripts/codex_heartbeat.py`. This file is local and ignored by git because
  it changes every heartbeat cycle.

## Update Rules

1. Record only facts, explicit preferences, or high-confidence repeated
   patterns.
2. Include the date when adding durable memory.
3. Do not store API keys, tokens, cookies, personal identifiers beyond what is
   already present in trusted local paths, or copied chat transcripts.
4. When a remembered item becomes stale, mark it as superseded instead of
   deleting context that may explain earlier behavior.
