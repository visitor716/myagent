# Sync Shared Skills

Sync common, cross-harness user skills between `~/.codex/skills`, `~/.claude/skills`, and Hermes' `~/.hermes/skills/myagent` target without overwriting unmanaged conflicts.

## When to Use

- The user says `同步skill`, `同步技能`, `sync skill`, `sync skills`, `share skills`, or asks Codex and Claude Code to share the same user skills.
- You need to make a newly added user skill available in the opposite harness.
- You need Hermes to see a myagent-managed skill such as `daily-report-table`.
- You need to refresh Codex wrappers for Claude flat `*.md` skills.
- The user says `同步配置`, `sync configs`, `备份配置`, `恢复配置`, or asks to sync Claude Code / Codex settings between runtime and myagent.
- The user asks to back up Codex runtime experience, Codex prompts, agents, rules, memories, or reusable Codex configuration into `configs/codex/`.

## What Counts as a Common Skill

- Directory skills with `SKILL.md` that are not system/runtime-only orchestration flows.
- Claude flat skills like `review.md` or `typecheck.md` can be imported into Codex by generating a managed wrapper directory with `SKILL.md`.
- Skip `.system`, `learned`, imported/generated buckets, and OMX/Codex runtime workflows such as `autopilot`, `team`, `ralph`, `ultrawork`, `cancel`, `trace`, `worker`, `hud`, and `wiki`.

## Direction

- In Codex sessions, sync from `~/.codex/skills` to `~/.claude/skills`:

```bash
bash scripts/sync_shared_skills.sh --source codex
```

- To make Codex skills visible to Hermes, sync from `~/.codex/skills` into Hermes' myagent skill bucket:

```bash
bash scripts/sync_shared_skills.sh --source codex --target hermes
```

- In Claude Code sessions, sync from `~/.claude/skills` to `~/.codex/skills`:

```bash
bash scripts/sync_shared_skills.sh --source claude
```

- Preview first when the user asks for inspection-only behavior or when conflicts are likely:

```bash
bash scripts/sync_shared_skills.sh --source codex --dry-run --verbose
bash scripts/sync_shared_skills.sh --source claude --dry-run --verbose
bash scripts/sync_shared_skills.sh --source codex --target hermes --dry-run --verbose
```

## Workflow

1. Resolve the source and target roots:
   - Codex user skills: `${CODEX_HOME:-~/.codex}/skills`
   - Claude user skills: `~/.claude/skills`
   - Hermes myagent user skills: `~/.hermes/skills/myagent`
2. Collect candidate skills from the source side:
   - directory skills: `<name>/SKILL.md`
   - Claude flat skills: `<name>.md`
3. Filter out non-common skills using the script's exclusion rules.
4. Sync safely:
   - directory skill -> create a symlink in the target root when no unmanaged conflict exists;
   - Claude flat skill -> create or update a managed Codex wrapper directory `<name>/SKILL.md`.
   - Hermes target -> create symlinks under `~/.hermes/skills/myagent/<name>` so Hermes' normal local skill scanner can discover them.
5. Never overwrite or delete unmanaged targets. Report conflicts instead.
6. Print a concise summary of created, updated, unchanged, skipped, and conflicting entries.

## Conflict Rules

- If the target already points at the same source, leave it alone.
- If the target was previously generated by this skill, update it in place.
- If the target is unmanaged and different, skip it and report the conflict.
- Do not remove existing files or symlinks unless the user explicitly asks.

## Configs Sync

In addition to skills, this script can sync Claude Code and Codex configuration files between runtime directories and the myagent source-of-truth repository.

### Backup runtime configs to myagent

```bash
bash scripts/sync_shared_skills.sh --configs backup
```

Copies `~/.claude/settings.json` and `~/.codex/config.toml` to `/home/zhanxp/projects/myagent/configs/`.
**After backup, review and sanitize sensitive data (API keys, tokens) before committing.**

### Restore configs from myagent to runtime

```bash
bash scripts/sync_shared_skills.sh --configs restore
```

Restores configs from myagent to runtime directories. Creates timestamped backups of existing runtime configs.

### Validate config formats

```bash
bash scripts/sync_shared_skills.sh --configs validate
```

Validates JSON/TOML format of the stored config templates.

### Backup reusable Codex runtime experience

```bash
bash scripts/sync_shared_skills.sh --configs codex-runtime-backup
```

This delegates to `/home/zhanxp/projects/myagent/configs/sync.sh codex-backup-runtime`.
It writes the reusable Codex runtime snapshot to:

```text
/home/zhanxp/projects/myagent/configs/codex/runtime/
```

Included:
- `AGENTS.md`, `config.toml`, `hooks.json`, `version.json`
- `agents/`, `prompts/`, `rules/`, `memories/`, `skills/`
- `README.md` and `inventory.txt`

Excluded by design:
- `auth.json`, `history.jsonl`, logs, sessions, shell snapshots, SQLite state/cache
- expired OAuth authorize URL allow-rules in `rules/default.rules`

Use this for AI workflow experience preservation. Do not treat it as a full
runtime clone, because secrets and high-churn local state are intentionally
excluded.

## Verification

- After changing the sync logic, run:

```bash
# Skills sync dry-run
bash scripts/sync_shared_skills.sh --source codex --dry-run --verbose
bash scripts/sync_shared_skills.sh --source claude --dry-run --verbose
bash scripts/sync_shared_skills.sh --source codex --target hermes --dry-run --verbose

# Configs sync dry-run
bash scripts/sync_shared_skills.sh --configs backup --dry-run
bash scripts/sync_shared_skills.sh --configs restore --dry-run
bash scripts/sync_shared_skills.sh --configs validate
bash scripts/sync_shared_skills.sh --configs codex-runtime-backup
```

- If this skill itself needs to be callable from Claude Code, ensure `~/.claude/skills/sync-shared-skills` exists, preferably as a symlink to the Codex skill directory.

- Prefer the script over ad-hoc shell commands so filtering, wrapper generation, and conflict handling stay consistent.
